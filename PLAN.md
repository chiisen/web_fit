# Fit 運動與生活紀錄平台 - 專案規劃文件

## 一、專案概述

### 1.1 專案名稱
**Fit - 運動與生活紀錄平台 DEMO**

### 1.2 專案目標
建立一個整合多種運動數據來源的雲端平台，提供個人、健身房、企業健康管理的完整運動紀錄與分析解決方案。

### 1.3 專案定位
- **個人用戶**：個人運動數據追蹤、健康管理、長期趨勢分析
- **健身房**：會員管理、團體訓練監控、教練端分析工具
- **企業**：員工健康管理、團隊健康推動、健康數據統計

---

## 二、平台定位與核心價值

### 2.1 目標用戶群
1. **個人運動愛好者**
   - 跑步、騎車、游泳、重訓等各類運動者
   - 需要長期追蹤與分析的運動數據

2. **健身房與教練**
   - 需要監控學員訓練狀態
   - 提供專業的訓練分析與建議

3. **企業健康管理**
   - 推動員工健康活動
   - 統計團隊健康數據

### 2.2 數據來源整合
- **穿戴裝置**：智慧手錶、手環（心率、步數、睡眠）
- **手機應用**：GPS 定位、運動記錄 App
- **IoT 健身器材**：跑步機、飛輪、重訓器材（功率、阻力、次數）

## 三、技術架構

### 3.1 前端技術選型

#### 核心技術
- **HTML5**：語義化標籤、結構化內容
- **Tailwind CSS**：響應式設計、快速開發
- **Chart.js**：數據視覺化圖表

- **無建置流程**：使用 CDN 引入，快速部署

- **語義化標籤**：使用 `<header>`, `<main>`, `<section>`, `<footer>` 等標籤

## 四、功能模組設計

### 4.1 運動資料收集模組

#### 4.1.1 跑步配速
- **數據項目**：距離、時間、平均配速、最快配速、海拔變化
- **視覺化**：配速曲線圖、海拔剖面圖
- **分析功能**：配速區間分析、PB 比較

#### 4.1.2 騎乘功率
- **數據項目**：功率輸出、踏頻、速度、距離、爬升
- **視覺化**：功率曲線、功率區間分布
- **分析功能**：FTP 分析、功率區間時間統計

#### 4.1.3 重訓肌力
- **數據項目**：動作名稱、重量、組數、次數、休息時間
- **視覺化**：肌力成長曲線、訓練量統計
- **分析功能**：1RM 預估、肌肉群訓練分布

#### 4.1.4 游泳泳姿
- **數據項目**：泳姿類型、距離、時間、划水次數、SWOLF
- **視覺化**：泳姿效率分析、速度曲線
- **分析功能**：泳姿效率比較、技術指標

#### 4.1.5 其他運動數據
- **有氧卡路里**：運動消耗卡路里計算
- **心率監測**：即時心率、心率區間分析
- **睡眠品質**：深度睡眠、淺眠、REM 階段
- **步數統計**：每日步數、活動量追蹤

### 4.2 雲端同步模組

#### 功能特點
- **跨裝置同步**：手機、平板、電腦數據同步
- **長期儲存**：歷史數據永久保存
- **自動備份**：定期自動備份數據
- **數據匯出**：支援 CSV、JSON 格式匯出

### 4.3 數據分析與視覺化

#### 4.3.1 心率區間分析
- **區間分類**：恢復區、有氧區、燃脂區、無氧區、最大區
- **視覺化**：柱狀圖顯示各區間時間分布
- **應用**：訓練強度評估、恢復建議

#### 4.3.2 速度分析
- **時間序列**：每日/每週/每月平均速度趨勢
- **視覺化**：折線圖顯示速度變化
- **應用**：進步追蹤、訓練效果評估

#### 4.3.3 瓦特功率分析
- **功率指標**：平均功率、最大功率、標準化功率
- **視覺化**：功率曲線、功率區間分布
- **應用**：訓練強度監控、FTP 測試

#### 4.3.4 步頻分析
- **步頻指標**：平均步頻、步頻變化
- **視覺化**：步頻曲線、步頻分布
- **應用**：跑步效率優化

#### 4.3.5 象限分析
- **分析維度**：強度 vs 效率
- **視覺化**：散點圖顯示訓練數據點
- **應用**：訓練品質評估、優化建議

### 4.4 團體訓練系統

#### 4.4.1 多人即時數據收集
- **即時監控**：顯示所有成員當前訓練狀態
- **數據項目**：心率、功率、速度、訓練強度
- **視覺化**：成員列表、即時數據面板

#### 4.4.2 教練端分析
- **訓練監控**：即時查看所有學員狀態
- **數據分析**：訓練強度分布、完成度統計
- **互動功能**：發送提醒、調整訓練計畫

### 4.5 健康管理

#### 4.5.1 體脂肪
- **數據追蹤**：體脂肪率、變化趨勢
- **健康範圍**：根據年齡、性別提供健康範圍參考
- **視覺化**：進度條、趨勢圖

#### 4.5.2 肌肉率
- **數據追蹤**：肌肉率、肌肉量變化
- **目標設定**：個人目標設定與追蹤
- **視覺化**：進度條、成長曲線

#### 4.5.3 水分率
- **數據追蹤**：身體水分率、每日變化
- **健康範圍**：理想水分率範圍提示
- **視覺化**：進度條、每日記錄

#### 4.5.4 睡眠品質
- **數據項目**：總睡眠時間、深度睡眠、淺眠、REM
- **品質評分**：睡眠品質分數
- **視覺化**：睡眠階段圖、品質趨勢

### 4.6 群組系統

#### 4.6.1 成員運動數據分析
- **統計功能**：群組總運動時數、平均完成率
- **排名系統**：成員排名、團隊排名
- **視覺化**：群組活動趨勢圖、成員貢獻度

#### 4.6.2 球隊健康推動
- **功能**：球隊成員運動數據統計
- **應用**：訓練計畫執行率、團隊目標達成
- **視覺化**：團隊數據面板、排名顯示

#### 4.6.3 企業健康管理
- **功能**：員工健康數據統計
- **應用**：部門健康分數、活動達成率
- **視覺化**：企業健康儀表板、部門比較

## 五、API 設計

### 5.1 運動紀錄 API

#### 5.1.1 建立運動紀錄
```
POST /api/v1/activities
Request Body:
{
  "user_id": "string",
  "activity_type": "running|cycling|swimming|strength|cardio",
  "start_time": "ISO 8601",
  "end_time": "ISO 8601",
  "data": {
    "distance": number,
    "duration": number,
    "pace": number,
    "power": number,
    "heart_rate": {
      "avg": number,
      "max": number,
      "zones": object
    },
    "calories": number,
    "steps": number
  }
}

Response:
{
  "id": "string",
  "status": "success",
  "created_at": "ISO 8601"
}
```

#### 5.1.2 查詢運動紀錄
```
GET /api/v1/activities
Query Parameters:
  - user_id: string
  - activity_type: string (optional)
  - start_date: ISO 8601 (optional)
  - end_date: ISO 8601 (optional)
  - page: number (optional)
  - limit: number (optional)

Response:
{
  "data": [
    {
      "id": "string",
      "activity_type": "string",
      "start_time": "ISO 8601",
      "duration": number,
      "distance": number,
      "calories": number
    }
  ],
  "pagination": {
    "page": number,
    "limit": number,
    "total": number
  }
}
```

#### 5.1.3 更新運動紀錄
```
PUT /api/v1/activities/:id
Request Body:
{
  "data": object
}

Response:
{
  "id": "string",
  "status": "success",
  "updated_at": "ISO 8601"
}
```

#### 5.1.4 刪除運動紀錄
```
DELETE /api/v1/activities/:id

Response:
{
  "status": "success"
}
```

### 5.2 群組管理 API

#### 5.2.1 建立群組
```
POST /api/v1/groups
Request Body:
{
  "name": "string",
  "type": "team|company|gym",
  "description": "string",
  "owner_id": "string"
}

Response:
{
  "id": "string",
  "name": "string",
  "type": "string",
  "created_at": "ISO 8601"
}
```

#### 5.2.2 加入群組
```
POST /api/v1/groups/:id/members
Request Body:
{
  "user_id": "string"
}

Response:
{
  "status": "success",
  "member_id": "string"
}
```

#### 5.2.3 查詢群組成員
```
GET /api/v1/groups/:id/members

Response:
{
  "data": [
    {
      "user_id": "string",
      "name": "string",
      "role": "member|admin",
      "joined_at": "ISO 8601"
    }
  ]
}
```

#### 5.2.4 查詢群組統計
```
GET /api/v1/groups/:id/statistics
Query Parameters:
  - start_date: ISO 8601 (optional)
  - end_date: ISO 8601 (optional)

Response:
{
  "total_members": number,
  "total_activities": number,
  "total_duration": number,
  "average_completion_rate": number,
  "ranking": number
}
```
