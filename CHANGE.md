### 任務 #2：調整成員運動數據分析圖表大小
**執行時間**：第一次優化  
**任務描述**：縮小「成員運動數據分析」圖表的顯示大小，使其不超過半個頁面

#### 問題描述：
- 成員運動數據分析圖表在頁面上占據過大空間
- 影響整體頁面布局與視覺平衡

#### 解決方案：
1. **移除固定高度屬性**
   - 原本：`<canvas id="groupActivityChart" height="100"></canvas>`
   - 修改：移除 `height="100"` 屬性

2. **添加容器高度限制**
   - 在圖表容器 div 上添加 `max-h-64` class（最大高度 256px）
   - 確保圖表不會超過容器高度

3. **調整 Chart.js 配置**
   - 將 `maintainAspectRatio` 從 `false` 改為 `true`
   - 設置 `aspectRatio: 3`（寬高比 3:1，讓圖表呈現更寬更扁的形狀）

#### 修改位置：
- **檔案**：`index.html`
- **區塊**：Groups Section 中的「成員運動數據分析圖表」區塊
- **行數範圍**：約 472-476 行（HTML）和 670-677 行（JavaScript 配置）

#### 修改前後對比：
```html
<!-- 修改前 -->
<div class="mt-12 bg-white p-8 rounded-xl shadow-md">
    <h3 class="text-2xl font-semibold mb-6 text-gray-800">成員運動數據分析</h3>
    <canvas id="groupActivityChart" height="100"></canvas>
</div>

<!-- 修改後 -->
<div class="mt-12 bg-white p-8 rounded-xl shadow-md">
    <h3 class="text-2xl font-semibold mb-6 text-gray-800">成員運動數據分析</h3>
    <div class="max-h-64">
        <canvas id="groupActivityChart"></canvas>
    </div>
</div>
```

```javascript
// 修改前
options: {
    responsive: true,
    maintainAspectRatio: false,
    // ...
}

// 修改後
options: {
    responsive: true,
    maintainAspectRatio: true,
    aspectRatio: 3,
    // ...
}
```

#### 結果：
- ✅ 圖表高度限制在 256px 以內
- ✅ 圖表呈現更寬更扁的比例（3:1）
- ✅ 不影響其他功能與布局
- ✅ 保持響應式設計
